╔══════════════════════════════════════════════════════════════════════════════╗
║  STEP 1 — Import flows into ORCE via the admin API                         ║
╚══════════════════════════════════════════════════════════════════════════════╝

Option A — kubectl port-forward (works without Ingress):

  kubectl -n {{ .Release.Namespace }} port-forward svc/{{ .Values.orce.serviceName }} 1880:{{ .Values.orce.port }} &

  kubectl get configmap {{ include "facis-ai-insight-ui.flowsConfigMapName" . }} \
    -n {{ .Release.Namespace }} \
    -o jsonpath='{.data.flows\.json}' | \
    curl -s -X POST http://localhost:1880/flows \
         -H 'Content-Type: application/json' \
         -H 'Node-RED-Deployment-Type: full' \
         -d @-

Option B — exec into the ORCE pod (no network access required):

  ORCE_POD=$(kubectl get pods -n {{ .Release.Namespace }} \
    -l app={{ .Values.orce.deploymentName }} \
    -o jsonpath='{.items[0].metadata.name}')

  kubectl exec -n {{ .Release.Namespace }} $ORCE_POD -- \
    sh -c 'curl -s -X POST http://localhost:1880/flows \
           -H "Content-Type: application/json" \
           -H "Node-RED-Deployment-Type: full" \
           -d "$(cat /data/flows.json)"'

  # Or copy the flows file into the pod first:
  kubectl cp flows/flows.full.json \
    {{ .Release.Namespace }}/$ORCE_POD:/tmp/flows.full.json
  kubectl exec -n {{ .Release.Namespace }} $ORCE_POD -- \
    curl -s -X POST http://localhost:1880/flows \
         -H "Content-Type: application/json" \
         -H "Node-RED-Deployment-Type: full" \
         -d @/tmp/flows.full.json

╔══════════════════════════════════════════════════════════════════════════════╗
║  STEP 2 — Mount UI static files into ORCE                                  ║
╚══════════════════════════════════════════════════════════════════════════════╝

Add the following volume and volumeMount to the ORCE Deployment
(kubectl edit deployment {{ .Values.orce.deploymentName }} -n {{ .Release.Namespace }}):

  volumes:
    - name: ai-insight-ui-files
      configMap:
        name: {{ include "facis-ai-insight-ui.uiFilesConfigMapName" . }}

  volumeMounts:            # inside the orce container
    - name: ai-insight-ui-files
      mountPath: /data/uibuilder/ai-insight/src
      readOnly: true

After saving, ORCE will restart and serve the Vue.js dashboard at:
  http://<orce-host>:{{ .Values.orce.port }}/ai-insight/
